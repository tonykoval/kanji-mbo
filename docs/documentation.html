Kanji MBO

A rule-based engine for categorizing and grouping Japanese kanji by shared components, readings, and semantic meaning.

Python 3 pandas openpyxl DisjointSet

What It Does

Kanji MBO reads kanji data from Excel spreadsheets and applies a multi-rule categorization algorithm to organize kanji into meaningful groups. The grouping is based on:

  • Keywords — semantic categories like "numbers", "colors", "family"
  • Stems — shared radical/component groupings
  • On'yomi readings — kanji sharing Chinese-origin pronunciations
  • Visual similarity — kanji sharing structural components

Two Pipelines

Categorization (src/) Frequency (src/freq/)
Purpose Full rule-based grouping into named categories (numbers, colors, body parts, etc.) Simplified grouping by shared component and on'yomi, ordered by frequency
Input 1500 or 2250 kanji spreadsheet with keyword, stem, and special lists 1,200 kanji spreadsheet (single sheet)
Rules 7-rule priority chain with SRL ordering and queue system Component matching + on'yomi filtering
Output Named groups with subgroup breakdowns (console) Numbered frequency-sorted list (output.txt)

Installation

Prerequisites

  • Python 3.8 or higher
  • pip package manager

Setup

1. Clone the repository

git clone https://github.com/your-username/kanji-mbo.git
cd kanji-mbo

2. Create a virtual environment

python -m venv venv

# Windows
venv\Scripts\activate

# macOS/Linux
source venv/bin/activate

3. Install dependencies

pip install -r requirements.txt

Data Files

Place your Excel data files in the excel/ directory. The project expects:

  • 1500 KANJI COMPONENTS - ver. 1.3.xlsx — primary dataset for the categorization pipeline
  • 2250 KANJI COMPONENTS - ver. 1.0.xlsx — expanded dataset (optional)
  • 1,200 KANJI.xlsx — dataset for the frequency pipeline (placed in src/freq/)
Note
The Excel files are the primary data source and must be present before running the scripts. They are not included in the repository.

Quick Start

Running the Categorization Pipeline

# Navigate to the src directory
cd src

# Run the main categorization
python main.py

This reads the Excel file, applies all 7 categorization rules to each kanji, and prints the results to the console.

Running the Frequency Pipeline

# Navigate to the frequency directory
cd src/freq

# Run the frequency grouping
python main.py

Output is written to src/freq/output.txt with kanji sorted by frequency rank.

Categorizing a Single Kanji

To test categorization of a specific kanji, uncomment the relevant lines in src/main.py:

# In src/main.py, uncomment:
algorithm.categorize_kanji(
    algorithm.read_kanji_char("麻", source),
    categorization,
    source
)

Switching Datasets

To use the 2250 kanji dataset instead of 1500, change the filename in src/main.py:

# Change from:
source = algorithm.read_excel("../excel/1500 KANJI COMPONENTS - ver. 1.3.xlsx")

# To:
source = algorithm.read_excel("../excel/2250 KANJI COMPONENTS - ver. 1.0.xlsx")

Adjusting Log Level

import logging

# Detailed rule-by-rule logging
algorithm.set_logging_level(logging.INFO)

# Only warnings and errors
algorithm.set_logging_level(logging.WARNING)

Categorization Pipeline

The main pipeline (src/main.py) processes each kanji through a 7-rule priority chain to assign it to a named group.

Processing Flow

Read Excel → Init Categorization → For each kanji: → Rule 1: Keyword? yes → Assign to keyword group → Rule 2: Stem? yes → Assign to stem group (first position) → Special? yes → Assign to special group → Rule 3: Crown? yes → Queue or Rule 4 → Rule 4: On'yomi? yes → Queue or Rule 5 → Rule 5: Stem var?yes → Assign to stem variation group → Rule 6: Visual? yes → Queue by visual similarity → Rule 7: Other → Assign to "79 other" Resolve queues → DisjointSet merge → Print results

Queue System

When a kanji references another kanji with higher SRL that hasn't been categorized yet, it enters a queue. The queue maps reference characters to lists of waiting kanji.

Example: On'yomi Queue

Consider processing 清 (sei/clear) before 青 (sei/blue):

  1. 清 has component2 = 青, and they share on'yomi セイ
  2. 青 has higher SRL, so 清 enters the queue: queue["青"] = [清]
  3. When 青 is later categorized into "03 colors", the queue is flushed — 清 joins the same group

DisjointSet Resolution

After all kanji are processed, remaining queue entries are resolved using a DisjointSet (Union-Find) data structure. This merges transitive relationships:

# If A references B, and B references C,
# DisjointSet ensures A, B, C all end up in the same group
ds = DisjointSet()
for key in categorization.result:
    for kanji in categorization.result[key]:
        ds.union(kanji.char, key)

Frequency Pipeline

The frequency pipeline (src/freq/) is a simpler alternative that groups kanji by shared components and on'yomi, then orders them by frequency rank.

How It Works

  1. Read — Load 1,200 kanji from Excel
  2. Match components — For each kanji, find others sharing component2
  3. Filter by on'yomi — Among matches, keep only those sharing a Chinese reading
  4. Set reference — The match with the lowest frequency rank (most common) becomes the reference
  5. Union-Find — Use DisjointSet to merge all transitive groups
  6. Sort and output — Print groups sorted by frequency

Example

Processing 中 (naka/middle):

-------------------------------------------
categorize: 中
components count: 2
components: 仲, 忠
onyomi filter count: 2
onyomi filter: 仲, 753 | 忠, 1182
RESULT: 中

中 shares component2 with 仲 and 忠. Both share on'yomi チュウ. Since 仲 has the lowest frequency (753), it could become the reference, but 中 itself is the one being categorized so it keeps its own reference.

Output

Written to src/freq/output.txt:

1: 人
2: 日
3: 一
4: 大
5: 出
6: 本
7: 中
...

Each kanji is numbered sequentially. Groups (sets of related kanji) appear consecutively, sorted by the lowest frequency rank within each group.

Section 2: Subgroups

Groups are broken down by the group field from the Excel data. Kanji without a group field are labeled "companion".

Tip
Subgroups like 1a, 1b, etc. represent sub-categories within a main group. "companion" kanji are those connected by reading or component similarity but not explicitly pre-assigned to a subgroup.

Algorithm Rules

Each kanji is evaluated against rules in priority order. The first matching rule determines its group.

Rule 1 — Keyword Match

Look up the kanji's keyword in the keyword.list sheet.

  • If type is MEAN: assign to the keyword's group
  • If type is OTHER: assign to "79 other"
Example
百 (hundred) has keyword "hundred" → maps to "01 numbers"

Rule 2 — Stem Match

Look up the kanji's character in the stem.list STEM KANJI column.

  • If type is STEM: assign to the stem's group and place it first in the group
Example
水 (water radical) is a stem kanji → placed first in "08 Wednesday" (water group)

Special Rule

If the kanji appears in special.list, it is assigned to the "special" group and its key is registered in the queue.

Rule 3 — Crown Component

Check if any other kanji has this kanji as its component2. Among those matches, filter by shared on'yomi.

  • If matches share on'yomi: tag with CROWN_TAG, set type to VR
  • If this kanji's SRL is higher: fall through to Rule 4
  • Otherwise: queue under the highest-SRL match
Example
青 (blue) appears as component2 of 晴, 清, 請, 靖, 精 — all share on'yomi セイ. 青 gets CROWN_TAG.

Rule 4 — On'yomi Cluster

Find kanji sharing the same component2 or whose character equals this kanji's component2. Filter by shared on'yomi.

  • If matches found: queue under the highest-SRL match or fall through to Rule 5
  • If no matches: fall through to Rule 5

Rule 5 — Stem Variation

Check if any of the kanji's components (1, 2, or 3) appear as stem variations in stem.list (columns STEM 1–6).

  • Priority: component1 (3) > component2 (2) > component3 (1)
  • If found: assign to the stem's group with type MEAN or FORM (based on SRL)
  • If not found: fall through to Rule 6

Rule 6 — Visual Similarity

Three-level fallback looking for component-sharing kanji:

  1. Find kanji sharing any of components 1, 2, or 3 with this kanji's character
  2. If empty: find kanji sharing the same component2
  3. If empty: find a kanji whose character equals this kanji's component1

Matches are queued by SRL. If nothing matches, fall through to Rule 7.

Rule 7 — Other (Fallback)

Assign to the "79 other" catch-all group.

API Reference

Key functions in src/algorithm.py.

Data Loading

read_excel(filename) → Source

Reads an Excel workbook and returns a Source with four DataFrames.

source = algorithm.read_excel("../excel/1500 KANJI COMPONENTS - ver. 1.3.xlsx")

read_kanji(row) → Kanji

Converts a DataFrame row into a Kanji dataclass instance.

read_kanji_char(char, source) → Kanji

Looks up a single kanji by character from the source DataFrame.

kanji = algorithm.read_kanji_char("青", source)

read_kanji_dataframe(dataframe) → List[Kanji]

Converts an entire DataFrame into a list of Kanji objects.

Categorization

init_categorization(source) → Categorization

Initializes the categorization with empty groups from keyword.list and stem.list, plus "other" and "special" groups. Populates the queue from special.list.

categorization = algorithm.init_categorization(source)

categorize_kanji(kanji, categorization, source)

Runs the 7-rule priority chain on a single kanji, modifying the categorization in place.

for i in range(len(source.df_kanji)):
    row = algorithm.read_kanji(source.df_kanji.loc[i])
    algorithm.categorize_kanji(row, categorization, source)

categorize_queue(categorization)

Resolves all queued kanji using DisjointSet and merges them into the result groups. Call this after all kanji have been processed.

algorithm.categorize_queue(categorization)

Lookup Functions

find_keyword(kanji, source) → Optional[str]

Returns the group name if the kanji's keyword exists in keyword.list, else None.

find_stem(kanji, source) → Optional[str]

Returns the group name if the kanji's character is a stem in stem.list, else None.

find_special(kanji, source) → Optional[str]

Returns "special" if the kanji exists in special.list, else None.

find_stem_variations(component, source, priority) → Optional[Stem]

Searches stem.list columns STEM 1–6 for the given component. Returns a Stem with group and priority, or None.

find_kanji_on_reading(cluster, kanji) → List[Kanji]

Filters a list of kanji to only those sharing at least one on'yomi reading with the given kanji.

Utility Functions

add_to_queue(kanji, ref_char, categorization)

Adds a kanji to the queue under the given reference character.

append_categorization(category, kanji, is_first, categorization)

Adds a kanji to a result group. If is_first=True, the kanji is inserted at position 0 (used for stem kanji). Also flushes any queued kanji waiting for this character.

set_logging_level(state)

Sets the module logger level (logging.INFO, logging.WARNING, etc.).

Glossary

TermDescription
SRL Stroke Radical Level. A priority value assigned to each kanji. Higher SRL means the kanji "leads" a group — lower-SRL kanji are queued to follow it.
On'yomi (音読み) Chinese-origin reading of a kanji. Kanji sharing on'yomi are often related and grouped together. Delimited by 、 in the data.
Kun'yomi (訓読み) Native Japanese reading of a kanji.
Component A structural part of a kanji. Each kanji can have up to 5 components (component1 through component5). Component2 is the most important for grouping.
Stem A root kanji or component that defines a group. Stem kanji are placed first in their group and matched via stem.list.
Crown Tag A tag (CROWN_TAG) added when a kanji is used as component2 by other kanji that share its on'yomi. Indicates the kanji "crowns" a reading cluster.
Companion A kanji grouped with a primary kanji through reading or component similarity, but without an explicit subgroup assignment.
Queue A temporary holding area for kanji whose reference kanji (higher SRL) hasn't been categorized yet. Resolved after all kanji are processed using DisjointSet.
DisjointSet A Union-Find data structure used to merge transitive group relationships during queue resolution.
VR A kanji type indicating it was grouped by on'yomi Voiced Reading match.
MEAN A kanji type indicating it was categorized by keyword meaning or stem meaning.
FORM A kanji type for stem-variation matches with SRL=1, emphasizing structural form over meaning.
Frequency Rank How common the kanji is in everyday use. Lower number = more common. Used by the frequency pipeline for ordering.
MBO Meaning Based Ordering — the organizing principle of grouping kanji by semantic and phonetic relationships.
